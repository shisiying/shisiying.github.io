优点:
1.职责清晰 真实的角色就是实现的业务逻辑，不用关心其他非本指责的事务
2.高扩展性 具体主题角色不能怎么变化，只要它实现了接口，代理类完全可以在不做任何修改的情况下使用
3.智能化 动态代理是最好的体现，即在运行阶段才指定实际代理对象
