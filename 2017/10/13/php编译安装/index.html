php编译安装

linux下使用GNU的AUTOCONF和AUTOMAKE产生的程序的安装步骤

  • ./configure是用来检测你的安装平台的目标特征的。比如它会检测你是不是有CC或GCC，并不是需要CC或GCC，它是个shell脚本。

  • make是用来编译的，它从Makefile中读取指令，然后编译。

  • make install是用来安装的，它也从Makefile中读取指令，安装到指定的位置。

解压以及预编译

tar zxvf php-7.0.24.tar.bz2

解压之后进行预编译

configure，这一步一般用来生成 Makefile，为下一步的编译做准备，你可以通过在 configure 后加上参数来对安装进行控制，比如代码:./configure –prefix=/usr 意思是将该软件安装在 /usr 下面，执行文件就会安装在 /usr/bin （而不是默认的 /usr/local/bin),资源文件就会安装在 /usr/share（而不是默认的/usr/local/share）

cd php-7.0.24

执行 

./configure --prefix=/usr/local/php

表示将php安装在上述目录下，conf放置配置文件

编译

make

安装到指定目录下

make isntall
